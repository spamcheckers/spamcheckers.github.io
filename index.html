How it works

From submission to verdict in milliseconds

The engine runs every layer in sequence. A single layer reaching a hard-block threshold stops processing immediately.

01

Email is submitted

User types into any form field. The real-time validator fires after 600ms of inactivity — or server-side on form submission.

↓
02

Whitelist check

Trusted domains and previously approved emails skip all layers instantly.

↓
03

6 layers run in parallel

Each layer scores 0–100. Weighted totals are summed. Any layer hitting a hard-block threshold forces an immediate block verdict.

↓
04

Verdict: Allow / Flag / Block

Below 30 = allowed. 30–59 = flagged (suspicious). 60+ = blocked. Every result is logged with full reason breakdown.

↓
✓

Reputation updated

The engine records the email and IP in its local memory table. Future checks for the same address score higher automatically.

Detection engine

Six independent layers, one confident verdict

Each layer is independently configurable. You can disable layers, adjust weights, and set hard-block thresholds from the settings page.

S

Syntax & Pattern Analysis

15% weight

Detects keyboard walks (qwerty123), gibberish local parts, all-numeric addresses, excessive repeating characters, suspicious TLDs, and known placeholder domains. Catches test@testing.com with an instant hard block.

D

Disposable Domain Detection

25% weight

Checks the domain against a database of 500+ known throwaway providers — Mailinator, Guerrilla Mail, Temp-Mail, YOPmail, and more. The list auto-updates weekly via WP-Cron.

M

MX Record Validation

20% weight

Queries DNS for the domain's mail exchange records. A domain with no MX or A record cannot receive email — ever. Score ≥ 80 triggers an immediate hard block regardless of the overall weighted total.

V

SMTP Handshake Verification

25% weight

Opens a real TCP connection to the mail server (EHLO → MAIL FROM → RCPT TO) and checks whether the specific mailbox exists. No email is ever sent. Score ≥ 50 hard-blocks.

B

Behavioral Analysis

10% weight

Checks honeypot field fill state, form completion time, and referrer headers. Bots that complete forms in under a second or fill hidden fields get an immediate hard block.

R

Reputation Memory

5% weight

Maintains a local SQLite-style table of past flagged emails and IPs. Repeat offenders score higher on every subsequent attempt. Completely private — all data stays on your server.
